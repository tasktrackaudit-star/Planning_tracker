⚠️ Fix Admin Access — Supabase SQL Required

Your Supabase profiles table has a recursive RLS policy that is blocking role reads. Copy the SQL below and run it in your Supabase Dashboard → SQL Editor, then click Refresh. 

-- Step 1: Remove the recursive policy (this causes the infinite loop)
DROP POLICY IF EXISTS "Admins read all profiles" ON profiles;

-- Step 2: Ensure safe self-read policy exists
DROP POLICY IF EXISTS "Users read own profile" ON profiles;
CREATE POLICY "Users read own profile" ON profiles
  FOR SELECT USING (auth.uid() = id);

-- Step 3: Allow users to update their own profile
DROP POLICY IF EXISTS "Users update own profile" ON profiles;
CREATE POLICY "Users update own profile" ON profiles
  FOR UPDATE USING (auth.uid() = id);

-- Step 4: Set your email as admin (change to your email if different)
UPDATE profiles
  SET role = 'admin'
  WHERE email = 'nikunjkoshiya29@gmail.com';

-- Step 5: Verify it worked
SELECT id, email, role FROM profiles
  WHERE email = 'nikunjkoshiya29@gmail.com';
After running the SQL: Come back here and click the ↻ Refresh button next to your role, or click your name at the bottom-left of the sidebar.
